# Security and Privacy

## Credentials Management
- All sensitive credentials are stored in `.env` files
- `.env` files are gitignored to prevent credential exposure
- Use `.env.example` as template for configuration
- Never commit API keys or passwords to version control

## API Key Security
- Gemini API keys should be kept private
- Use environment variables for all sensitive configuration
- Rotate API keys regularly
- Monitor API usage for unauthorized access

## Account Safety
- Use dedicated test accounts when possible
- Never share account credentials
- Enable two-factor authentication where available
- Monitor account activity for unauthorized access

## Responsible Usage
- Respect OGame Terms of Service
- Use rate limiting to avoid server overload
- Focus on educational and research applications
- Avoid disruptive or unfair gameplay practices
